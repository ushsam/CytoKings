5.1 ACCURACY METRICS:
------------------------------------------------------------------------------------------
Accuracy = (TP + TN) / Total
  Example: 60% means 60 out of 100 correct
  ⚠ Misleading if classes are imbalanced!
  For your data (50/50 split): 60% is slightly better than random (50%)

When to use:
  ✓ Use for balanced datasets (50/50 males/females)
  ✗ Don't use for imbalanced (e.g., 95% males/5% females)

5.2 PRECISION & RECALL:
------------------------------------------------------------------------------------------
Precision = TP / (TP + FP)
  'Of all predictions I made for Males, how many were correct?'
  High precision = Few false alarms

Recall = TP / (TP + FN)
  'Of all actual Males, how many did I catch?'
  High recall = Few missed cases

Precision-Recall Trade-off:
  Medical test: High recall (catch all sick people, few false negatives)
  Spam filter: High precision (reduce spam, few false positives)

5.3 F1-SCORE:
------------------------------------------------------------------------------------------
F1 = 2 × (Precision × Recall) / (Precision + Recall)
  Balanced score between precision and recall
  Range: 0 to 1 (1 = perfect)
  F1 = 0.60 means 'fair' prediction ability

5.4 ROC-AUC:
------------------------------------------------------------------------------------------
AUC = Area Under ROC Curve
  Probability that model ranks random positive higher than random negative
  Range: 0.5 (random) to 1.0 (perfect)
  Your result: ~0.60 = slightly better than random

Interpretation:
  0.5-0.6 = Poor
  0.6-0.7 = Fair
  0.7-0.8 = Good
  0.8-0.9 = Excellent
  0.9-1.0 = Outstanding
